# AYG Project Overview

> Version 1.0

---

# Vision

誰もが、正しいエネルギー管理を通じて理想の身体を実現できる世界をつくる。

世の中にはダイエットや筋トレに関する情報が溢れている。

しかし、多くの人は

- 何を信じればいいのか分からない
- 何をどれだけ食べればいいのか分からない
- どれだけ運動すればいいのか分からない

という状態である。

AYGは、科学的根拠に基づいたエネルギー管理によって、その迷いをなくす。

---

# Mission

食事・運動・体重を横断して管理し、

「今日、自分は何をすれば目標に近づけるのか」

を誰でも分かるようにする。

複雑な計算や知識はすべてAYGが行い、

ユーザーは迷わず行動できる状態を提供する。

---

# Product

AYG（サービス名：カロナビ予定）は、

食事・運動・体重・目標を統合して管理する

**エネルギー管理アプリケーション**である。

一般的な食事管理アプリでも、

筋トレ記録アプリでもない。

エネルギー収支を中心に、

ユーザーの目標達成をサポートする

**意思決定支援アプリ**である。

---

# Core Value

AYGが提供する価値は

**エネルギー管理による意思決定支援**

である。

例えば、

・今日はあと何kcal食べられるか

・今日はこれだけ食べたから何をどれだけ運動すればよいか

・今日運動したからあとどれだけ食べられるか

・今日は食べ過ぎたので何日かけて調整すればよいか

・今のペースで目標日に間に合うか

など、

ユーザーが毎日悩む判断を、

科学的根拠に基づいて数値化し、

分かりやすく提示する。

---

# Philosophy

## 1. Evidence First

すべての計算・ロジックは

可能な限り科学的根拠を基に構築する。

トレンドや個人の経験ではなく、

再現性のある知見を優先する。

---

## 2. Energy First

AYGの中心は

エネルギー管理である。

体重管理は

摂取エネルギーと消費エネルギーの

長期的な収支によって決まる。

すべての機能は、

この考え方を中心に設計する。

---

## 3. Simplicity Outside

ユーザーが見る画面は

可能な限りシンプルにする。

内部では複雑な計算を行っていても、

ユーザーには

「今日はあと〇〇kcal食べられます。」

「今日は〇〇分歩けば調整できます。」

のように、

行動だけが伝わればよい。

---

## 4. Complexity Inside

内部ロジックは

可能な限り正確に設計する。

入力が少なくても、

可能な限り現実に近い計算を行う。

---

## 5. Input Minimum

入力負担は最小限。

ユーザーが継続できることを最優先とする。

入力が面倒で続かないサービスは

価値を提供できない。

---

## 6. Trust

AYGは

ユーザーの身体を扱うプロダクトである。

そのため、

分からないことは

分からないと伝える。

断定できないものは断定しない。

信頼を最優先とする。

# Project Principle

このプロジェクトでは、

**ユーザーに機能を提供すること**

が目的ではない。

ユーザーに

**目標を達成してもらうこと**

が目的である。

そのため、

機能を増やすことではなく、『この機能は本当にユーザーの目標達成率を高めるか』を、すべての意思決定の基準とする。
